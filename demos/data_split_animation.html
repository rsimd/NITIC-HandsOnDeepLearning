training data / validation data / test data の関係

Iris 150件を，コード中の比率に従って training data 90件，validation data 30件，test data 30件に分ける

1. 全データを用意する

X, y: 150 samples
0150 samples
all data150件 / 100%
分割前の全データ この時点では，まだ学習用・検証用・最終評価用の役割は決まっていない。

2. training data と test data に分割する

train_test_split(..., test_size=0.2)
training 80%test 20%
training data120件 / 80%
test data30件 / 20%
training data モデル作成のために使ってよいデータ。この中から validation data を作る。
test data 最終評価専用のデータ。実験（パラメータの学習）にこれを使うとカンニング（leakage）になるので，無いものとして扱おう。

3. training data から validation data を作る

train_test_split(training data, test_size=0.25)
training 60%validation 20% / test 20%
training data90件 / 60%
validation data30件 / 20%
test data30件 / 20%
training data パラメータ更新に使う。`optimizer.step()` が直接見るデータ。
validation data 擬似的な test data として，学習中の性能確認やハイパーパラメータ調整に使う。
test data 全ての実験が終わった後，最後に1回だけ最終評価に使う。
