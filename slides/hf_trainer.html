Trainer の使い方

PyTorch の学習ループを「決まった部品」に分けて渡す

Trainer とは？

  • forward → loss → backward → optimizer.step の流れを管理
  • 評価・ログ出力・保存を定期的に実行
  • GPU / CPU の切り替えや batch 処理も任せられる

このスライドで作るもの

  • Iris データセットを Trainer 用 Dataset にする
  • 普通の MLP を Trainer 用モデルで包む
  • accuracy・TrainingArguments・Trainer を接続する

1Trainer 用 Dataset を作る

Trainer は Dataset から dict を受け取ります。今回のキーは features と labels です。

IrisDataset
class IrisDataset(Dataset[dict[str, torch.Tensor]]):
    """Iris の特徴量とラベルを Trainer 用の辞書で返す Dataset."""

    def __init__(self, features: np.ndarray, labels: np.ndarray) -> None:
        self.features = torch.tensor(features, dtype=torch.float32)
        self.labels = torch.tensor(labels, dtype=torch.long)

    def __len__(self) -> int:
        return len(self.labels)

    def __getitem__(self, index: int) -> dict[str, torch.Tensor]:
        return {
            "features": self.features[index],
            "labels": self.labels[index],
        }

ここで重要なこと

  • __len__：データ数を返す
  • __getitem__：1 サンプルを辞書で返す
  • features は float32
  • labels は分類用なので long
💡
キー名が model.forward の引数名と対応するfeatures と labels を返すので、後で作る forward(features, labels) にそのまま渡せます。

3Trainer 用モデル：logits と loss を返す

Trainer はモデルの forward が返す辞書を見ます。分類では logits と loss が重要です。

class ClassifierForTrainer(nn.Module):
    """Trainer でクラス分類を学習するためのモデル."""

    def __init__(self, net: nn.Module, loss_fn: nn.Module) -> None:
        super().__init__()
        self.net = net
        self.loss_fn = loss_fn

    def forward(
        self,
        features: torch.Tensor,
        labels: torch.Tensor | None = None,
    ) -> dict[str, torch.Tensor]:
        logits = self.net(features)
        output = {"logits": logits}
        if labels is not None:
            output["loss"] = self.loss_fn(logits, labels)
        return output

なぜ包むのか？

  • 普通の nn.Sequential は logits しか返さない
  • Trainer は学習時に loss を必要とする
  • そこで、元のネットワークに loss 計算を追加する

labels がない場合

推論や一部の評価では labels がないこともあるので、そのときは logits だけ返せる形にしています。

4MLP・損失関数・最適化手法を定義

ここでは Iris の 4 特徴量から 3 クラス分類する MLP を作ります。

# ニューラルネットワークの構造を定義する
net = nn.Sequential(
    nn.Linear(4, 16),
    nn.ReLU(),
    nn.Linear(16, 8),
    nn.ReLU(),
    nn.Linear(8, 3),
)

# 損失関数を定義する
loss_fn = nn.CrossEntropyLoss()

# 最適化手法を定義する
optimizer = torch.optim.SGD(net.parameters(), lr=0.05)

# Trainer 用のモデルを作成する
trainer_model = ClassifierForTrainer(net, loss_fn)
部品意味
nn.Linear(4,16)Iris の 4 個の特徴量を入力
nn.Linear(8,3)3 クラス分のスコアを出力
CrossEntropyLoss多クラス分類でよく使う損失関数
SGD重みを更新する方法
ClassifierForTrainerTrainer が扱える返り値に整えるラッパー
⚠️
learning rate が 2 箇所にあるこのコードでは optimizer の lr=0.05 が実際に使われます。後で optimizers=(optimizer, None) として明示的に渡すためです。

5評価指標 accuracy を計算する

Trainer の評価結果 EvalPrediction から logits と正解ラベルを取り出し、accuracy を返します。

# Trainer の評価結果から accuracy を計算する関数を定義する
def compute_iris_metrics(eval_pred: EvalPrediction) -> dict[str, float]:
    """Trainer の評価結果から accuracy を計算する."""
    logits = eval_pred.predictions
    labels = eval_pred.label_ids
    predictions = np.argmax(logits, axis=1)
    return {"accuracy": accuracy_score(labels, predictions)}

処理の流れ

  • logits：各クラスのスコア
  • np.argmax(..., axis=1)：一番スコアの高いクラスを予測ラベルにする
  • accuracy_score：正解ラベルと予測ラベルの一致率を計算

6TrainingArguments：学習の設定を書く

エポック数、バッチサイズ、ログ頻度、評価頻度、保存先などをまとめて指定します。

    num_train_epochs=80,

    per_device_eval_batch_size=50,
    learning_rate=0.01,
    logging_strategy="epoch",
    eval_strategy="epoch",
    seed=SEED,
    use_cpu=not torch.cuda.is_available(),
)

per_device_eval_batch_size評価時のバッチサイズ
引数説明

7Trainer を作成して部品を接続する

ここまで作った「モデル・設定・データ・評価関数・optimizer」を Trainer に渡します。

iris_trainer = Trainer(
    model=trainer_model,
    args=training_args,
    train_dataset=trainer_train_dataset,
    eval_dataset=trainer_val_dataset,
    compute_metrics=compute_iris_metrics,
    optimizers=(optimizer, None),
)
引数渡すもの
modellogits と loss を返すモデル
args学習設定
train_dataset学習データ
eval_dataset検証データ
compute_metrics評価指標を計算する関数
optimizersoptimizer と scheduler

8train() を呼ぶと学習が始まる

最後は train() を呼び出すだけです。

train_output = iris_trainer.train()

Trainer が内部で行うこと

  • Dataset から batch を作る
  • model.forward を呼んで logits / loss を得る
  • backward して optimizer で重みを更新
  • epoch ごとに評価・ログ出力を行う

まとめ

Trainer は魔法ではなく、毎回書く学習ループを共通化したものです。

  • Dataset：features と labels を返す
  • Model：logits と loss を返す
  • Metrics：評価結果から accuracy を返す
  • TrainingArguments：学習の設定を書く
